GenAI Mt Acknowledgement

Overview

Throughout the design and development of the "Recipe Explorer" website, I actively integrated Generative AI (GenAI) tools such as ChatGPT and selectively applied Machine Translation (MT) tools when testing multilingual phrases (e.g., marking non-English food names using lang attributes). This section details how these tools enhanced productivity, creativity, and technical implementation — while critically reflecting on limitations, ethical safeguards, and future professional applications.

⚙️ Strategic Use of GenAI in Implementation

1. Technical Implementation Support

GenAI was used to modularize monolithic CSS files into mobile-first, component-based styles (e.g., newspaper&aboutus.css). It helped generate:

1. Responsive layout using @media (min-width: 768px)

2. Reusable style patterns (e.g., .subscribe-form, .about-icons-grid)

3. Visual consistency across pages using existing var(--color) tokens

GenAI guided the conversion of JavaScript logic into ES6 modules:

1. Created postFormData.js and getData.js for comment and form API interactions.

2. Helped modularize mobile and desktop menu toggles with reusable functions and event listeners.

3. Enhanced interactivity with scalable carousel/slider logic in the Showcase section.


2. Content Authoring and Enhancement

I used GenAI to draft and simplify instructional text on forms and buttons, such as:

Checkbox labels for privacy consent

Error-free placeholder text for input fields

Mobile-friendly form descriptions

GenAI helped optimize the semantic use of "section", "nav", and "main" elements, ensuring the site met accessibility and SEO best practices.


3. Accessibility Testing and Interpretation

I used GenAI to interpret WCAG 2.1 AA principles when IBM Equal Access Checker flagged a low-contrast warning (2.13:1).

GenAI helped me differentiate between decorative vs. functional text, confirming that the symbolic star rating didn’t require correction — a nuance I wasn’t initially aware of.

// showcaseSlider.js
export function initShowcaseSlider() {
const track = document.querySelector(".showcase-track");
const leftBtn = document.querySelector(".arrow.left");
const rightBtn = document.querySelector(".arrow.right");

let scrollIndex = 0;

rightBtn.addEventListener("click", () => {
const cards = track.children.length;
if (scrollIndex < cards - 3) scrollIndex++;
track.style.transform = `translateX(-${scrollIndex * 33.33}%)`;
});
leftBtn.addEventListener("click", () => {
if (scrollIndex > 0) scrollIndex--;
track.style.transform = `translateX(-${scrollIndex * 33.33}%)`;
});
}

// youMayLikeScroll.js
export function initYouMayLikeAutoScroll() {
const track = document.querySelector(".you-may-like-track");
let scrollAmount = 0;

setInterval(() => {
scrollAmount += 220; // or card width
if (scrollAmount >= track.scrollWidth - track.clientWidth) {
scrollAmount = 0;
}
track.scrollTo({ left: scrollAmount, behavior: "smooth" });
}, 4000);
}

🧠 Critical Evaluation of AI-Assisted Design

Significantly reduced time spent debugging layout issues.

Improved consistency in form behavior and styling across multiple pages.

Allowed me to focus more time on creative aspects(e.g., designing "Showcase", "Newest Uploads", and "You May Like This" layout variations).

Brainstormed alternative UI patterns and interaction ideas (e.g., healthy meter circle visual, hover-underline effects).

Provided prompt feedback when exploring how to structure recipe cards and transition from mockups to working HTML/CSS.

GenAI occasionally provided incomplete or outdated code patterns, such as:

- Missing aria-labels or unnecessary inline event handlers.

- Overly generic layouts that ignored the project’s color palette or spacing scheme.

Required manual alignment with design tokens (e.g., -primary-color) and removal of conflicting CSS declarations.

All generated content was reviewed, revised, and customized to suit the actual project context.

No direct copy-paste from AI-generated code; instead, I used it as a scaffold to test and refine my own implementation.

MT tools were used strictly for language labeling and basic comprehension, not for bulk translation or content generation.


📈 Future Plans for Professional GenAI/MT Use

Use GenAI as a creative and debugging assistant, not as a code replacement.

Cross-check AI recommendations with official documentation and community best practices, especially in accessibility and security domains.

Develop custom prompt templates for tasks such as generating WCAG summaries, scaffolding semantic HTML, or producing visual design alternatives.

Leverage GenAI in collaborative settings for documentation, UI/UX ideation, and client proposal drafting.

Use MT tools to test multilingual support and communicate basic cultural cues when designing inclusive web interfaces.

Maintain transparency in all GenAI-influenced work.

Respect authorship and originality in team environments.

Continue building awareness of biases or hallucinations in GenAI outputs and use domain knowledge to verify facts and correctness.


🎯 Final Reflection

GenAI and MT tools were instrumental in enhancing both the technical and creative dimensions of my website project. They enabled rapid prototyping, improved code structure, and supported my learning process without compromising academic integrity. However, their use also reinforced the need for critical evaluation, manual testing, and thoughtful application in professional practice. Moving forward, I see these tools not as shortcuts, but as intelligent collaborators — ones that must be guided by ethical design values, user-centered thinking, and responsible engineering judgment.
