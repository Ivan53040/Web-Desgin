Website Structure

Page implemented:

Homepage, Culture Showcase, Recipe & Cooking, Recipe example, Healthy Living, Sharing Area, Contact Us

- Core landing page introducing the platform and its purpose.

- Designed to feel warm, welcoming, and community-driven.

- Includes Recipe Example, Sharing Area

- Only the latest 5 comments are shown dynamically.

- Sharing Area let users share their food phots with comments

- Includes: Culture Showcase, Recipes & Cooking, Healthy Living, Contact Us

- These provide static cultural, health, and culinary content for users.

🔒 Why These Pages Are Static

- Current State: Static

- Reason: Stories are collected from the internet and presented purely as informational content.

- No dynamic user content is involved.

- Current State: Mostly static

- Reason:
1. Recipes require extensive structured data: ingredients, steps, nutrition, video links, etc.
2. The API used only supports limited content, so dynamic interaction is limited to a quick comment section (see: `recipe_example.html` using `postFormData.js` and `getData.js`)
3. To maintain content quality, all recipe posts require admin verification before publishing.

- Current State: Static

- Reason:
1. Provides curated health information.
2. The "Latest" section is administrator-controlled to avoid misinformation and ensure credibility.

In the A3 proposal, a dedicated Forum Page was planned. However, during implementation, this was streamlined into an embedded comment section on the Recipe Example page to simplify navigation and reduce cognitive load. This decision maintained user engagement goals while reducing redundancy. Additionally, while A3 included a dynamic news feed for Healthy Living, API limitations meant this feature is planned for future integration using verified health news APIs.

🌐 Technical, Ethical & Accessibility Considerations

1. All user-facing forms (newsletter, comment, submit recipe) only request essential data (e.g., name and email).

2. No sensitive data is collected. The site avoids intrusive features like third-party tracking or data monetization.

3. Submit Recipe requires admin verification to prevent spam/junk data.

All design decisions were guided by three core ethical principles: minimizing data collection, ensuring content integrity through moderation, and maximizing user safety and clarity through accessible interaction.


♿ Integrated Accessibilitys

Accessibility was evaluated using IBM Equal Access Checker with a focus on the Recipe Example page and general templates. Each page was tested for contrast, semantics, keyboard navigation, and structural clarity. Feedback was iteratively addressed during styling and layout design.

The site design and functionality were tested for accessibility using IBM Equal Access Checker, with the Recipe Example page serving as a representative sample.

Key findings included:

  • Color contrast ratios met WCAG 2.1 AA standards.
  • All interactive elements were keyboard navigable.
  • Semantic HTML was used to enhance screen reader compatibility.

🔎 Accessibility Breakdown (WCAG 2.1 AA)

1.1.1 Non-text Content: All images include meaningful alt text (e.g., recipe images).

1.2.2 Captions (Prerecorded): YouTube videos used include captions.

1.3.1 Info & Relationships: Semantic HTML used (e.g., "table" for nutrition).

1.4.1 Use of Color: Allergen icons supplement color labels.

2.1.1 Keyboard: Entire website is operable via keyboard (Tab, Enter, Arrow keys).

2.2.1 Timing Adjustable: Embedded YouTube content has native pause controls.

2.3.1 Flash Threshold: No animations exceed 3 flashes/second.

2.4.2 Page Titled: Every page includes clear "title".

2.4.4 Link Purpose (In Context): All buttons have descriptive labels (e.g., “View Recipe” not “Check Out now”).

3.1.1 Language of Page: (html lang="en") set on every page.

3.1.2 Language of Parts: Non-English terms like かつ丼 are marked with lang="ja".

3.2.1 On Focus: No unexpected content changes; form expansions are predictable.

3.2.3 Consistent Navigation: Menus are consistent across the site.

3.2.5 Consistent Terminology: Ingredient names and headings are reused consistently.
