Ethics Analysis

Background

    The client wants to build a third-party newsletter subscription plug-in the website, which is provided by one of our funders. It requires the users provides the following information:

    • • Full name
    • • Email address
    • • Occupation
    • • Annual income
    • • Household size
    • • Sexuality
    • • Gender
    • • Spending habits
    • • Type of phone they currently use
    • • Monthly spend on dining out/takeaway food
    • • Number of credit cards they have with the credit limit on each card

    However, the plug-in is collecting data that is not appropriate and may violates the The ACS Ethical Framework, this page will have a discussion on the ethical concerns and provides solutions to it.

Ethical Concern

Transparency and User Consent(E1)

ACS Code of Professional Ethics
Honesty - (2.1a, 2.1b)

Users may not fully understand why personal data (such as sexuality, income, and credit card limits) is being collected. Lack of transparency could be seen as misrepresentation or dishonesty.

Data Security and User Protection(E2)

ACS Code of Professional Ethics
Trustworthiness - (2.2d, 2.2h)

Collecting these sensitive informations may increases the risk of data breaches. We should ensure the security of protecting user data.

Respect for User Privacy and Inclusivity(E3)

ACS Code of Professional Ethics
Respect for Others - (2.3.1c, 2.3.1d, 2.3.1g)

The form may collect unnecessary personal data or lack inclusive options for users with diverse identities.

Ethical Responsibility in ICT and Public Trust(E4)

ACS Code of Professional Ethics
Respect for the Profession - (2.3.2a, 2.3.2c, 2.3.2g)

We are risking the public trust in ICT system if we try to integrate an intrusive data-collection tool without questioning it's purpose.

Ethical Solution and Pratical solution

The following cards shows the ethical solution of the concern above and showing the practical solution while flipping the card.

Ensuring users fully understand what they are consenting to.(E1)

✅ Clear privacy notice to explain why the data is being collected.

Preventing misrepresentation by clearly stating the purpose of data collection.(E1)

✅ Checkbox consent for non-essential fields (e.g., "I agree to share my financial details for research purposes").

Upholding user autonomy by allowing opt-outs for sensitive data..(E1)

✅ Alternative subscription option** allowing users to sign up with only an email.

Protecting user data from unauthorized access, avoiding potential harm.(E2)

✅ Limited data retention—store data only as long as necessary(e.g., The data will be delete when user unsubscribe the newspaper plug-in).

Respecting user rights by minimizing unnecessary data collection.(E3)

✅ Data minimization principle (only collecting the necessary information such as email/contact number)

Prioritizing public interest over business demands.(E4)

✅ Advocate with the client to reduce the amount of unnecessary data being collected.

After analysing the plug-in, it is suggested to reduce some of the none necessary data to be collected:
  • • Full name(or Neckname)
  • • Email address
  • • Occupation
  • • Annual income
  • • Household size
  • • Sexuality (become optional)
  • • Gender (become optional)
  • • Spending habits (become optional)
  • • Type of phone they currently use
  • • Monthly spend on dining out/takeaway food (become optional)
  • • Number of credit cards they have with the credit limit on each card
