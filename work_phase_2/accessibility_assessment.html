Accessibility Assessment

Perceivable

Definition:Users with visual impairments or hearing disabilities require alternative ways to access content.


1.1. Text Alternatives


1.1.1 Non-text Content
Provide recipe photos, ingredient lists, cooking steps.

1.2 Time-Based Media


1.2.2 Captions(Prerecorded)
Offer closed captions for all cooking videos to support users who are deaf or hard of hearing. 1.2.5 Audio Description (Prerecorded)
Include audio narration of key visual elements(e.g. importance steps) in cooking videos.

1.3 Adaptable


1.3.1 Info and Relationships
Use semantic HTML (headings, lists, tables) so screen readers can interpret recipe structures.

1.4 Distinguishable


1.4.1 Use of Color
Ensure color is not the only method to convey information (e.g., allergens marked with symbols, not just red text).
1.4.10 Adjustable Playback Speed
Allow users to slow down or speed up cooking tutorial videos for better comprehension.

Operable

Definition:Users with motor impairments or those who rely on keyboard navigation need a site that can be accessed without a mouse.


2.1 Keyboard Accessible


2.1.1 Keyboard
Ensure all interactive elements (recipe steps, video controls, navigation) are keyboard-accessible.(e.g. space to play/stop, right key to fast forward)
2.1.2 No Keyboard Trap
Users should be able to navigate in and out of every element using only a keyboard.(e.g. backspace to back to previous page)

2.2 Enough Time


2.2.1 Timing Adjustable
Allow users to pause, extend, or disable time limits on recipe steps or instructional videos. 2.2.2 Pause, Stop, Hide
Provide pause/resume controls for auto-scrolling recipe pages or autoplay videos.

2.3 Seizures and Physical Reactions


2.3.1 Three Flashes or Below
Ensure that flashing content (animations, video transitions) does not exceed three flashes per second.

2.4 Navigable


2.4.2 Page Titled
Ensure recipe pages have clear and descriptive titles for searchability.(e.g. tags catagory)
2.4.4 Link Purpose (In Context)
Avoid vague links like "Click Here"—use descriptive text like "View Full Recipe".
2.4.11 Provide Alternative Navigation Methods
Implement voice search or shortcut keys for easy recipe lookup.

Understandable

Definition:Users with cognitive disabilities or non-native language speakers need clear, structured instructions for recipes.


3.1 Readable


3.1.1 Language of Page
Use language attributes (lang="en") to help screen readers detect the correct language. 3.1.2 Language of Parts
Mark foreign food names (e.g., lang="ja" for "寿司" (sushi)) so screen readers pronounce them correctly.

3.2 Predictable


3.2.1 On Focus
Avoid sudden content changes when a user clicks on a form or a button.
3.2.3 Consistent Navigation
Ensure menus, sidebars, and page layouts remain consistent across recipes.
3.2.5 Consistent Terminology and Instructions
Use the same names for ingredients across all recipes (e.g., always use "cornflour" instead of "corn statch").

3.3 Input Assistance


3.3.1 Error Identification
Avoid sudden content changes when a user clicks on a form or a button.
3.3.2 Labels or Instructions
Ensure input fields (e.g., "Enter amount of flour") provide units of measurement(e.g. gam) for clarity.
3.3.6 Error Prevention
Ask users to confirm recipe submissions before posting to avoid accidental errors.
