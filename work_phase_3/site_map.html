Site Map

This is a site map for the project. It is a visual representation of the website's structure and content.

Inspiration

Dedicated to celebrating diverse food heritage (Client Needs, User Stories), using visuals and stories to evoke a sense of connection and authenticity (Visceral Design). Content is structured for easy navigation and includes accessibility features for media (Accessibility).

Highlights the source of ingredients and connects users to the land (User Stories, Client Needs).

Features diverse global cuisines, promoting cultural exchange (User Stories, Client Needs).

Focuses on preserving and sharing Australian culinary heritage (Client Needs, User Stories).


The core functionality for finding, sharing, and learning recipes (User Stories). Features like smart filters and clear instructions address user needs for simplicity and adaptability, built with accessibility in mind (Accessibility). The presentation aims to be inspiring and easy to follow (Visceral Design), supporting healthy and diverse cooking (Client Needs).

Central place to browse and find recipes (User Stories).

Helps users quickly narrow down recipe options based on various criteria (User Stories, Accessibility).

Enables users to share their own culinary creations, fostering community and diverse content (Client Needs, User Stories, Accessibility for input).

Provides visual, step-by-step guidance (User Stories), with captions and adjustable speed for accessibility (Accessibility).

Assists users in identifying and potentially locating ingredients (User Stories), improving the usability of recipes.


Provides spaces for users to connect, share experiences, and support each other (User Stories, Client Needs). Designed to feel welcoming and safe (Visceral Design), with features like forums and chat facilitating interaction, built with consideration for language barriers and navigation (Accessibility).

A place for personal narratives related to food and culture (User Stories, Client Needs).

Allows users to ask questions, share tips, and engage in conversations (User Stories, Accessibility for readability and posting).

Offers immediate communication for quick exchanges and connection (User Stories, Accessibility).

Facilitates in-person or virtual gatherings based on shared interests (User Stories, Client Needs).

Provides an overview of upcoming community activities (User Stories, Accessibility for navigation).

A personal space for users to log their cooking journey and share with others (User Stories).


Offers resources and recipes focused on health and nutrition (Client Needs, User Stories). Information is presented clearly and simply (Accessibility), aiming to make healthy eating approachable and enjoyable (Visceral Design).

Provides guidance and advice on nutritious diets (User Stories).

Offers strategies and recipes for planning healthy meals in advance (User Stories).

Provides details about the nutritional content of food and recipes (User Stories, Accessibility for clarity).

Helps users find healthier or more readily available ingredient alternatives (User Stories, Accessibility for clarity).


A personalized area for users to manage their activity on the platform (User Stories). Designed for ease of use and a sense of control, contributing to a positive user experience (Visceral Design, Accessibility for navigation and input).

Allows users to bookmark their favorite recipes and resources (User Stories).

Enables users to customize their presence and manage their information (User Stories, Accessibility for forms).


Provides information about the platform and the team (Client Needs, User Stories). Aims to build trust and transparency (Visceral Design) with clear contact information and accessible links (Accessibility).

Introduces the team and mission (Client Needs, User Stories).

Provides avenues for external connection (User Stories).

Offers a way for users to get in touch (User Stories, Accessibility for forms).

Shares the background and vision of the platform (Client Needs, User Stories).
